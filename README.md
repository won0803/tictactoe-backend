# 🎮 실시간 웹 게임 'TicTacToe (TTT)' 백엔드 API 서버

Spring Boot와 MyBatis를 활용하여 효율적인 MVC 패턴 기반으로 설계된 틱택토 게임 백엔드 서버입니다. 구조적인 패키지 분리를 통한 아키텍처 설계, 관계형 데이터베이스 정규화, 그리고 웹 보안 취약점 진단 및 UX 최적화에 초점을 맞춰 개발했습니다.

---

## 🛠 사용 기술
* Java 17 / Spring Boot
* MyBatis
* MySQL
* Gradle
  
---

## 🔥 핵심 구현 및 구조적 설계

### 1. MVC 아키텍처 기반의 엄격한 패키지 분리
* 효율적인 MVC 패턴 준수를 지향하며 게임(Game)과 사용자(User) 등 핵심 기능별로 계층을 나누어 독립적인 구조로 설계했습니다.
* 이러한 구조적 분리는 코드의 가독성을 높일 뿐만 아니라, 향후 특정 기능의 확장이나 유지보수 시 영향 범위를 최소화하여 시스템의 안정적인 운영을 가능하게 합니다.

### 2. 정규화된 데이터베이스 모델링 및 전적 관리
* 게임 및 유저 데이터를 관계형 데이터베이스(MySQL) 환경에 맞게 유연하게 정규화하여 관리합니다.
  * `T_USERS`: 보안을 고려한 유저 계정 정보 저장
  * `T_GAME`: 실시간 게임 보드 상태 및 진행 데이터 관리
  * `T_RANK`: 유저별 승/무/패 전적 합산 및 승률 기반의 랭킹 시스템 구축
* 로비 대시보드를 통해 활성화된 방의 상태(대기/진행)를 한눈에 파악하고, 마이페이지에서 상세 개인 전적을 확인할 수 있도록 지원합니다.

### 3. 계정 보안 및 핵심 게임 엔진 로직 구현
* **인증 시스템 고도화:** 이메일 기반 인증 코드 시스템을 도입하여 계정 찾기 보안성을 강화했으며, 비밀번호 해싱(Hashing)을 적용하여 유저의 개인정보를 안전하게 보호합니다. ID·이메일·닉네임의 실시간 유효성(중복) 검증 로직도 포함되어 있습니다.
* **승리 판정 알고리즘:** 3x3 2차원 배열을 순회하며 가로, 세로, 대각선의 동일 표식(O, X) 여부를 판별하는 `checkWin` 엔진 로직을 안정적으로 구현했습니다.
* **상태 무결성 검증:** 현재 보드 상태를 문자열 데이터로 변환하여 DB에 동기화하고, 각 턴마다 유저의 세션과 대조 검증하여 이미 선택된 칸 중복 선택 방지 및 턴 유효성 체크 등 정교한 게임 룰을 통제합니다.

## 🛠 트러블슈팅 및 UX 최적화

### 🚨 URL 파라미터 개인정보 노출 취약점 해결
* **문제 상황:** 비밀번호 찾기 및 이메일 인증 과정에서 사용자 이메일 등의 민감 정보가 URL 파라미터에 평문으로 노출되는 보안 취약점을 확인했습니다. 이대로 두면 브라우저 기록에 개인정보가 남거나 외부 유출 위험이 있었습니다.
* **해결 방안:** 클라이언트 단에서 `replaceState API`를 활용해 인증 직후 주소창에서 파라미터를 즉시 제거하도록 처리했으며, 필요한 인증 데이터는 `sessionStorage`로 은닉하여 안전하게 보존했습니다. 이를 통해 보안성 확보와 새로고침 시 데이터 연속성을 보장하는 UX 최적화를 동시에 달성했습니다.

### 🚨 파라미터 유실로 인한 서버 400 에러 방지
* **문제 상황:** 특정 페이지 새로고침이나 비정상적인 접근 시, 필수 파라미터가 유실되어 서버 단에서 400 에러(Bad Request)가 발생하며 페이지가 튕기는 현상이 있었습니다.
* **해결 방안:** 컨트롤러 레벨에서 `@RequestParam(required=false)` 설정을 활용해 파라미터 존재 여부와 상관없이 예외를 부드럽게 흡수하도록 조정했으며, 내부 데이터 복구 및 방어 로직을 추가하여 데이터 유실 환경에서도 서버가 안정적으로 흐름을 제어할 수 있도록 보완했습니다.
