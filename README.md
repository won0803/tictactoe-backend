# 🎮 실시간 웹 틱택토 게임 백엔드

Spring Boot와 웹소켓(WebSocket)을 공부하면서 유저 간 실시간 매칭과 동기화가 가능한 게임 서버를 직접 구현해 본 프로젝트입니다. 단순히 기능을 만드는 것을 넘어, 웹 환경에서 발생하는 실시간 통신의 한계와 보안 요소를 고민하며 개발했습니다.

---

## 🛠 사용 기술
* Java 17 / Spring Boot / Spring Data JPA
* MySQL
* Spring WebSocket / STOMP
* Gradle

---

## 🔥 핵심 구현 및 고민한 점

### 1. 웹소켓(WebSocket)과 STOMP를 활용한 실시간 동기화
* 기존 HTTP 통신의 단방향 한계를 해결하기 위해 웹소켓을 도입했습니다.
* STOMP 프로토콜을 활용해 플레이어 간의 턴(Turn) 전환 데이터와 돌이 놓이는 위치 값을 실시간으로 오차 없이 주고받도록 동기화 환경을 구축했습니다.

### 2. 커스텀 인터셉터(Interceptor)를 통한 세션 및 보안 관리
* 실시간 연결이 맺어질 때 아무나 접근하지 못하도록 깃허브 로그인 정보나 인증 여부를 검증하는 웹소켓 인터셉터를 직접 구현했습니다.
* 비정상적인 접근을 연결 단계에서 차단하여 게임의 안정성과 보안성을 높였습니다.

### 3. 인메모리 기반 매칭 큐(Queue) 설계
* 게임방 진입 시 유저들을 대기시키고 빠르게 대전 상대를 매칭해 주기 위해, 메모리 상에서 동작하는 매칭 큐와 대기방 메커니즘을 직접 구현했습니다.
* 상대가 매칭되면 즉시 실시간 게임방이 동적으로 생성되도록 처리했습니다.
